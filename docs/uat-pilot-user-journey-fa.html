۱ — این برنامه دقیقاً چیست؟

هم‌سازمان (logestic) یک نرم‌افزار برای مدیریت حمل بار معدن است — مثلاً معدن تفتان می‌خواهد ۱۰ تن سنگ حمل شود:

  • کارفرما/معدن نیاز ثبت می‌کند.
  • مدیر عملیات راننده را روی آن نیاز می‌گذارد (dispatch).
  • راننده با اپ موبایل مسیر را طی می‌کند تا بار تحویل شود.
  • اپرator باسکول وزن خالی/پر را وارد می‌کند.
  • سیستم پول را بین مالک ناوگان، پلتفرم (۱٪) و صندوق تعاونی (سهم روستا) تقسیم می‌کند.
  • آخر ماه تسویه انجام می‌شود و خانوارها سهم خود را در اپ تعاونی می‌بینند.
شما در UAT چه می‌کنید؟ خودتان نقش هر شخص را یک‌بار بازی می‌کنید — مثل یک نمایش — تا ببینید همه مراحل درست کار می‌کند یا نه.

۲ — داستان یک بار حمل (قلب برنامه)

قبل از کلیک کردن، کل فلو را یک‌بار بخوانید. بعد در فاز ۲ همین را عملی انجام می‌دهید.

۱. کارفرما → «۱۰ تن سنگ می‌خواهم» (ثبت نیاز) ↓ ۲. مدیر عملیات → راننده را روی نیاز می‌گذارد (تخصیص / dispatch) ↓ ۳. راننده (اپ) → می‌پذیرد → می‌رود معدن → بار می‌زند → می‌آورد → تحویل می‌دهد ↓ ۴. اپرator باسکول (وب) → وزن خالی و پر را از نمایشگر باسکول می‌خواند و وارد می‌کند ↓ ۵. مدیر عملیات → باسکول را تأیید می‌کند → بار «تأییدشده» (VERIFIED) می‌شود ↓ ۶. سیستم → ~۹۹٪ برای مالک ناوگان · ~۱٪ پلتفرم · مبلغ تعاونی = تن × نرخ قرارداد ↓ ۷. آخر ماه → بستن ماه → صورت وضعیت → تأیید تعاونی + عملیات → قفل → پرداخت معدن → قفل تسویه ↓ ۸. مالک ناوگان و خانوار → کیف پول / سهم را می‌بینند
قبل از مرحله ۳: راننده باید در سیستم ساخته شده و احراز هویت (KYC)اش را مدیر تعاونی تأیید کرده باشد — وگرنه اپ اجازه مأموریت نمی‌دهد.

۵ — ورود، OTP، workspace

۵.۱ ورود

  1. صفحه login را باز کنید (وب یا اپ).
  2. شماره موبایل را بزنید → «دریافت کد».
  3. کد OTP را وارد کنید.
برای تست UAT (بدون SMS): روی پنل وب دکمه‌های ورود دمو هست — یا در اپ همان نقش را از لیست دمو انتخاب کنید. برای OTP بدون SMS: GET /api/auth/__dev/otp?mobile_number=... (فقط وقتی ENABLE_DEMO_LOGIN=true).

۵.۲ بعد از ورود — «workspace» یعنی چی؟

سیستم می‌پرسد روی کدام معدن و با چه نوع عضویت کار می‌کنید:

انتخابکی بزند
کار عملیاتی · معدن ۱ (TAFTAN)کارفرما، عملیات، راننده، باسکول (وب)، مالک ناوگان، ADMIN
عضویت تعاونی · معدن ۱مدیر تعاونی، خانوار، (KYC در اپ تعاونی)
workspace اشتباه = منو خالی یا خطا. logout کنید و دوباره workspace درست را بزنید.

۵.۳ خروج و عوض کردن نقش

برای تست هر نقش: خروج (منوی پنل وب · یا دکمه logout در app bar اپ‌های موبایل) → دوباره login با موبایل آن نقش (یا دکمه دمو). شما یک نفرید که همه نقش‌ها را پشت سر هم بازی می‌کنید.

۵.۴ صفحه انتخاب workspace — /workspace-select

  • بعد از OTP لیست membershipها نمایش داده شود
  • کار عملیاتی · معدن TAFTAN — برای ADMIN، EMPLOYER، OPS، DRIVER، FLEET، COOP_OPERATOR (باسکول)
  • عضویت تعاونی · معدن TAFTAN — برای COOP_ADMIN، HOUSEHOLD، COOP_OPERATOR (KYC موبایل)
  • انتخاب → redirect به home نقش

۵.۵ خانه پنل — /panel (همه نقش‌های وب)

  • حداکثر ۵ کارت میانبر — بدون کارت قفل
  • کلیک هر کارت → همان صفحه درست باز شود

    بعد از فاز ۲، این‌ها را برای پوشش کامل امتحان کنید.

    ۳.۱ احراز هویت (KYC)

    • خانوار: اپ تعاونی → ثبت‌نام چندمرحله‌ای → COOP_ADMIN در وب «تأیید هویت» → approve → wallet فعال
    • راننده: ADMIN ساخت → اپ «KYC در انتظار» → coop approve → مأموریت باز می‌شود
    • مالک ناوگان: درخواست KYC + خودرو → coop approve/reject
    • رد درخواست / درخواست اصلاح / تعلیق — هر کدام را یک‌بار بزنید

    ۳.۲ ساخت کاربر با «درخواست» (نه مستقیم ADMIN)

    • COOP_ADMIN → منو «ثبت کاربر جدید» → درخواست اپرator یا خانوار → ADMIN → «درخواست‌های کاربر» → تأیید → کاربر جدید login
    • OPERATION_ADMIN → همان مسیر → درخواست OPERATOR → ADMIN تأیید
    • یک درخواست را ADMIN رد کند — دلیل نمایش داده شود

    ۳.۳ کارفرما — فقط نیاز

    • ثبت نیاز جدید · مشاهده inbox · لغو نیاز PENDING
    • نباید بتواند dispatch یا تسویه ببیند (منو نباشد / 404)

    ۳.۴ اپ راننده — جزئیات

    • دکمه back در login → خروج از اپ (نه گیر کردن در splash)
    • حالت آفلاین: یک مرحله بدون اینترنت → بعد sync

    ۳.۵ اپ تعاونی — خانوار

    • تب‌ها: خانه · کیف پول · سهم · درخواست/اعتراض
    • ثبت اعتراض · مشاهده وضعیت تسویه

    ۳.۶ کار ساعتی (فاز ۲ — اختیاری)

    • EMPLOYER → ثبت نیاز → تب «ساعتی» (نه حمل)
    • OPERATOR (اپ community · workspace عملیاتی) → /hourly/start → START + GPS → /hourly/end → END
    • CONSULTANT (وب) → /panel/consultant/hourly → verify یا reject
    • CONSULTANT → /panel/payments مستقیم → 404 (فقط inbox ساعتی)

    ۳.۷ سناریوهای خاص (پوشش باگ)

    • HOLD باسکول: اختلاف وزن ≥۵٪ → PENDING_HOLD → مأموریت از تسویه حذف شود
    • نگهداری پرداخت: OPS → payments → hold/release روی mission_id
    • IBAN تکراری: دو خانوار با یک شبا → خطای validation
    • OTP unregistered: موبایل خارج از DB → «در سامانه ثبت نشده»
    • Community pool: OPS settlement → distribute روی pool SNAPSHOT_LOCKED (در صورت UI)

    ۳.۸ HOUSEHOLD روی وب (اختیاری)

    • Login HOUSEHOLD → workspace COMMUNITY → /panel/wallet
    • /panel/members — لیست اعضا · ثبت اعتراض روی یک عضو

    فاز ۴ قبل از رفتن به production

    • فاز ۲ (۱۵ قدم حمل) بدون blocker تمام شد
    • SMS واقعی: OTP با کاوه‌نگار روی یک موبایل واقعی تست شد (نه فقط دمو · __dev/otp در production خاموش است)
    • DNS + SSL: hamsahman.ir به سرور اشاره کند · HTTPS
    • APK production: base URL = https://hamsahman.ir (بدون /api)
    • کاربران واقعی با موبایل واقعی در ADMIN ساخته شدند (نه فقط 09000000*)
    • backup دیتابیس یک‌بار گرفته شد
    • تست خودکار: npm run test:infra-regression · test:pilot-taftan1 · test:e2e -- uat-haul (طبق چک‌لیست تحویل)
    • چک‌لیست فنی: uat-handover-checklist-fa.md
