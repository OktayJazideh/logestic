## ۱. فلسفه محصول (قفل شده)

> کاربر نباید «نقشه»، «قطب‌نما» یا «اسطرلاب» ذهنی برای پیدا کردن کار بسازد.  
> هر صفحه باید **بدون فکر کردن** جواب دهد: *الان کجام؟ · قدم بعدی چیه؟ · کدوم دکمه را بزنم؟*

این سند بر پایه اصول شناختی رایج (شناخت به‌جای یادآوری، یک CTA، پیش‌بینی‌پذیری مکان دکمه‌ها، معنی رنگی ثابت) و **حفظ چیدمان وایرفریم** (`docs/wireframes-review/wireframe Screenshots/`) نوشته شده است.

**خارج از scope این سند:** تغییر API/FSM/قوانین مالی — فقط لایه UI، copy، چیدمان، توکن بصری.

## ۲. اصول الزامی (همه صفحات)

### ۲.۱ بار شناختی صفر

| اصل | پیاده‌سازی |
|-----|------------|
| **یک کار اصلی در هر صفحه** | حداکثر **یک** دکمه Primary (پررنگ، تمام‌عرض پایین در موبایل) |
| **جای ثابت عناصر** | CTA اصلی همیشه پایین · عنوان بالا · وضعیت زیر عنوان · nav همان جا در همه اپ |
| **بدون منوی عمیق** | nav ≤ ۵ آیتم · بقیه «بیشتر» / تنظیمات |
| **بدون اصطلاح فنی در UI** | جدول §۴ — نه Provisioning، نه FSM، نه ledger |
| **شناخت > یادآوری** | هر action مهم: **آیکون + متن فارسی** (نه فقط آیکون) |
| **پیش‌بینی‌پذیری** | همان الگو در وب و موبایل: کارت bordered، استپر عمودی، badge وضعیت |

### ۲.۲ رنگ و مغز پیش‌فرض (همیشه همراه آیکون/متن)

| معنی | رنگ پیشنهادی | مثال copy |
|------|--------------|-----------|
| ادامه / تأیید / موفق | سبز (`#1B5E20` family) | «ادامه» · «ثبت شد» |
| خطر / رد / توقف | قرمز | «رد درخواست» · «لغو» |
| منتظر / هشدار | کهربایی | «در انتظار تأیید» |
| اطلاعات خنثی | آبی ملایم یا خاکستری تیره | «راهنما» |
| غیرفعال | خاکستری روشن | دکمه disabled + توضیح کوتاه چرا |

**ممنوع:** فقط رنگ برای انتقال معنی · کنتراست زیر WCAG AA · متن قرمز روی پس‌زمینه قرمز.

### ۲.۳ تایپوگرافی و لمس

| سطح | وب | موبایل (Flutter) |
|-----|-----|----------------|
| عنوان صفحه | 22–24px bold | 22sp |
| بدنه | ≥16px | ≥18sp |
| راهنما / زیرنویس | 14px muted | 14sp |
| دکمه اصلی | min-height 48px | min 56dp height · hit area 48dp |

### ۲.۴ خطا و خالی

- **خطا:** یک جمله ساده + **«چکار کنم؟»** (یک اقدام) — **بدون** `error_code` برای کاربر نهایی.
- **Loading:** «در حال بارگذاری…» + اسکلتون یا spinner مرکزی — نه صفحه سفید.
- **Empty:** «هنوز چیزی نیست» + یک CTA یا راهنمای یک خطی.

### ۲.۵ پیشرفت (مأموریت / فرم)

- «**مرحله ۲ از ۴**» همیشه visible.
- استپر **عمودی** مثل وایرفریم ۳: ✓ انجام‌شده · ● الان · ○ بعد · قفل برای آینده.
- FSM بک‌اند ۹ حالته عوض **نمی‌شود** — فقط لایه ۷ گام UI (جدول §۶).

## ۷. چک‌لیست «بدون فکر کردن» (UAT)

هر صفحه P1 — همه باید **بله** باشد:

1. آیا در ۳ ثانیه مشخص است «الان کجا» هستم؟
2. آیا **یک** دکمه اصلی واضح‌تر از بقیه است؟
3. آیا بدون آموزش قبلی می‌دانم قدم بعدی چیست؟
4. آیا متن‌ها فارسی محاوره‌ای‌اند (بدون انگلیسی در دکمه)؟
5. آیا خطا می‌گوید چکار کنم (نه فقط «خطا ۴۰۰»)؟
6. آیا دکمه‌های خطر (رد/لغو) از ادامه متمایزند (رنگ + متن)؟
7. آیا در موبایل با انگشت بزرگ قابل لمس است؟
8. آیا چیدمان شبیه وایرفریم همان اسکرین است (اسکرین‌شات diff)؟
9. آیا nav بیش از ۵ آیتم اصلی ندارد؟
10. آیا کاربر ناچار به حدس زدن آیکون تنها نشده؟
