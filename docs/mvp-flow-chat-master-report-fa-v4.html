۱. خلاصه یک‌صفحه‌ای

تخمین پوشش: Backend ~90% · Web ~75% · Driver ~85% · Community ~65% · Deploy واقعی ~15%

Freeze فاز ۱ (قفل): فقط حمل · Community = تن × مبلغ ثابت قراردادی · Operational = 99/1 · Pool مساوی snapshot · HSA Draft→Lock انسانی · پرداخت معدن فقط به حساب رسمی تعاونی · اعتراض ناشناس ممنوع · پلتفرم = زیرساخت (مثل اسنپ) نه مجری معدن.

Blocker اصلی UAT میدانی: deploy روی سرور + داده پایلوت کارفرما. باسکول = ورود دستی اپرator (Agent سخت‌افزار فاز بعد). کد P0 بسته: FIN-SEED-99-1 (99/1) + HOLD-SETTLEMENT-1.

تصمیم قفل باسکول: خرید/نصب Agent سخت‌افزار برای UAT اول لازم نیست. COOP_OPERATOR وزن را در پنل وب وارد می‌کند. scaffold در apps/weighbridge-agent/ برای بعد.

۴. Definition of Done — UAT تفتان (۱۰ گام)

  1. OTP ورود + انتخاب workspace عملیاتی/Community
  2. ثبت نیاز حمل کارفرما + dispatch دستی
  3. FSM راننده تا DELIVERED + geofence
  4. ثبت وزن باسکول — دستی COOP_OPERATOR در پنل (+ anomaly 5% → HOLD)
  5. تأیید OPERATION_ADMIN → VERIFIED + split 99/1
  6. Community contribution در pool (تن × نرخ قرارداد)
  7. صورت‌وضعیت Draft → Lock → settlement sheet
  8. مأموریت HELD در settlement نیست ✓
  9. Community app: ثبت خانوار + KYC
  10. Receipt PDF + audit log قابل ردیابی

۷. متن آماده برای کارفرما (کپی واتساپ/ایمیل)

چیزهایی که قفل شد

  • مدل مالی Dual Economy: کرایه 99/1 جدا از Community (تن × مبلغ ثابت قرارداد)
  • حمل-only فاز ۱ — ساعتی پیچیده فاز بعد
  • HSA: Draft اتومات + تأیید/قفل انسانی قبل از پرداخت معدن

چیزهایی که از شما لازم داریم (۱۱ مورد)

  1. عدد نهایی مبلغ Community به ازای هر تن (الان placeholder 400,000 ریال)
  2. نام رسمی تعاونی + IBAN حساب رسمی
  3. مسئول عملیات/تحویل + تاریخ هدف UAT
  4. لیست روستاهای مصوب (Excel یا نقشه)
  5. مشخصات باسکول (فقط اگر بعداً Agent خواستید — فعلاً دستی OK)
  6. لیست کاربران پایلوت (موبایل + نقش)
  7. تأیید برند «سهمان» و دامنه‌ها (title، SMS، legal)
  8. Dispatch پایلوت: دستی کافی است یا از روز اول auto؟
  9. تسویه هفتگی مالک: در UAT اول فعال شود؟
  10. Community Fund: ثبت «رسید بانک» در UI لازم است؟
  11. دسترسی سرور/DNS برای deploy (یا تأیید Staging ما)

پیشنهاد ما

  • اول Staging با SMS mock — بعد production با SMS واقعی
  • backup هفتگی Postgres از روز اول
  • UAT مالی کامل بعد از دریافت عدد قرارداد Community

پاسخ به نگرانی شفافیت تعاونی

سیستم audit log، صورت‌وضعیت قفل‌شده، و تقسیم مساوی snapshot دارد. رأی‌گیری/حکمرانی سنگین عمداً فاز بعد است تا پایلوت سریع و قابل اندازه‌گیری بماند.
