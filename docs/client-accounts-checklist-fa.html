راهنمای خرید سرویس‌ها — پلتفرم معدن (logestic)

برای کارفرما — بدون اصطلاحات سنگین. مشخص می‌کند چه بخرید، چه امکاناتی لازم است، و چه دسترسی‌هایی را به تیم فنی بدهید. از هر بخش یکی از پیشنهادها را انتخاب کنید کافی است.

حداقل برای روشن شدن واقعی سیستم: دامنه ایران + سرور/هاست ایران + پیامک (SMS) برای ورود با کد + کاربران با موبایل واقعی.
بقیه (Firebase، باسکول خودکار، بانک خودکار، …) را می‌شود بعداً اضافه کرد.
نقش شما: خرید، قرارداد، پرداخت دوره‌ای، تحویل پنل‌ها به تیم فنی
نقش تیم فنی: نصب، اتصال، تست، به‌روزرسانی نرم‌افزار
تحویل امن: یوزر/پسورد و API Key در یک فایل رمزگذاری‌شده یا تماس — نه پیام تصادفی واتساپ

اول بفهمید: «هاست وب» با «سرور» فرق دارد

این پلتفرم دو بخش دارد که روی اینترنت می‌آیند:

  • پنل وب — همان صفحه‌ای که ادمین، تعاونی، باسکول، کارفرما در مرورگر باز می‌کنند (مثل یک سایت).
  • موتور اصلی (API) — مغز سیستم: ورود با کد، مأموریت، پول، باسکول، تسویه. اپ موبایل راننده هم به همین وصل می‌شود.
  • دیتابیس — جایی که همه اطلاعات ماندگار ذخیره می‌شود (مثل یک صندوق فایل منظم، ولی حرفه‌ای‌تر).

هاست وب معمولی (همان چیزی که برای سایت شرکتی می‌خرند) فقط فایل‌های صفحه را نگه می‌دارد. به تنهایی برای این پروژه کافی نیست چون API و دیتابیس باید همیشه روشن باشند و برنامه Node.js اجرا شود.

سرور / VPS / پلتفرم ابری یعنی یک کامپیوتر در دیتاسنتر ایران که برنامه شما ۲۴ ساعته روی آن کار کند. معمولاً همان‌جا یا روی سرویس جدا، PostgreSQL هم می‌گذارند.

پیشنهاد عملی: یا همه‌چیز روی یک VPS (ایران‌سرور، پارس‌پک، …) یا روی سرویس آماده مثل لیارا (اپ + دیتابیس + فایل استاتیک پنل). پنل وب را می‌توان روی همان سرور گذاشت، یا جدا روی CDN ابر آروان برای سرعت بیشتر — هر دو درست است.

شکل پیشنهادی آدرس‌ها (با تیم فنی هماهنگ کنید):
panel.name-mine.ir → پنل وب برای کارمندان
api.name-mine.ir → ارتباط اپ موبایل و منطق سیستم

۱ — دامنه اینترنتی (.ir)

چه امکاناتی لازم است؟

  • دامنه .ir (ترجیحاً برای کاربران و اعتماد داخل ایران)
  • امکان تنظیم رکورد DNS (حداقل دو زیردامنه: panel و api)
  • امکان فعال‌کردن گواهی امنیتی HTTPS (قفل سبز در مرورگر) — معمولاً تیم فنی با Let's Encrypt رایگان انجام می‌دهد

پیشنهادها — یکی کافی است

گزینه الف — ایران‌نیک رایج

خرید دامنه .ir از nic.ir / نمایندگان ایران‌نیک. مناسب اگر فقط دامنه می‌خواهید و سرور جدا دارید.

به تیم فنی بدهید: ورود پنل DNS یا دسترسی برای ساخت رکوردها

گزینه ب — پارس‌پک / ایران‌سرور / لیارا

خیلی وقت‌ها دامنه را همان‌جا که سرور می‌خرید ثبت می‌کنند؛ راحت‌تر برای یک فاکتور.

گزینه ج — ابر آروان (فقط اگر پنل را روی CDN آن‌ها بگذارید)

دامنه را می‌توانید آنجا هم مدیریت کنید؛ معمولاً دامنه را از نماینده .ir می‌خرند و DNS را به آروان می‌دهند.

هزینه تقریبی دامنه .ir: حدود چند صد هزار تا چند میلیون تومان در سال (بسته به نام دامنه).

پیشنهادها — یکی را انتخاب کنید

گزینه ۱ — لیارا (Liara.ir) پیشنهاد برای شروع ساده

مناسب وقتی: نمی‌خواهید خودتان سرور لینوکس را مدیریت کنید؛ می‌خواهید با پنل فارسی اپ را بالا بیاورید.

چه بخرید (نام سرویس‌ها در پنل لیارا — معادل):

  • برنامه Node.js — پلن با حداقل ۱ GB RAM برای API (با رشد → ۲ GB)
  • دیتابیس PostgreSQL — پلن Managed Database با حداقل ۱ GB RAM (با داده زیاد → ۲–۴ GB)
  • استاتیک / Static App — برای فایل‌های پنل وب (حجم کم)
  • یا یک VPS لیارا با ۴–۸ GB RAM اگر ترجیح می‌دهید همه‌چیز یک‌جا باشد

امکانات لازم در پنل: Deploy از Git یا آپلود، متغیر محیطی (Environment Variables)، لاگ، پشتیبان خودکار دیتابیس (Backup)

به تیم فنی بدهید: دعوت به تیم در پنل لیارا یا API Token + دسترسی ساخت اپ/دیتابیس

گزینه ۲ — ایران سرور (IranServer.com) VPS کلاسیک

مناسب وقتی: می‌خواهید کنترل کامل داشته باشید یا تیم فنی به SSH عادت دارد.

چه بخرید:

  • Cloud VPS یا سرور مجازی ایران
  • پکیج پیشنهادی شروع: ۴ vCPU / ۸ GB RAM / ۸۰ GB SSD (نام دقیق پلن در سایت را ببینید — معادل همین اعداد مهم است)
  • سیستم‌عامل: Ubuntu 22.04 LTS (رایگان — هنگام خرید انتخاب شود)
  • IP ثابت ایران + امکان باز کردن پورت ۸۰ و ۴۴۳

به تیم فنی بدهید: IP سرور + user/password SSH (یا کلید SSH) — ترجیحاً user محدود، نه root مستقیم

گزینه ۳ — پارس‌پک (ParsPack.com)

مشابه ایران‌سرور — VPS ابری با پنل. همان مشخصات ۴–۸ GB RAM را بگیرید.

به تیم فنی بدهید: مانند گزینه ۲

گزینه ۴ — ترکیبی (حرفه‌ای‌تر برای ترافیک زیاد)

  • API + دیتابیس: VPS ایران‌سرور یا لیارا (۸ GB RAM)
  • پنل وب (فقط فایل‌های سایت): ابر آروان — CDN + فضای استاتیک (سرعت باز شدن پنل در شهرهای مختلف بهتر می‌شود)

دامنه: panel → آروان ، api → VPS

به تیم فنی بدهید: پنل آروان + پنل VPS

هاست اشتراکی معمولی (cPanel با PHP) را به تنهایی نخرید — برای این پروژه کافی نیست. اگر چنین هاستی دارید، فقط می‌تواند پنل استاتیک را نگه دارد؛ API باید جای دیگر باشد.

دیتابیس PostgreSQL — جدا بگویید

اگر از VPS استفاده می‌کنید، دیتابیس روی همان سرور نصب می‌شود (تیم فنی). اگر لیارا، سرویس PostgreSQL Managed بخرید راحت‌تر است (پشتیبان خودکار).

فضاشروع: ۱۰–۲۰ GB کافی؛ با سال‌ها داده معدن → ۵۰ GB+
پشتیبانBackup روزانه خودکار — حتماً از فروشنده بپرسید
دسترسیفقط از داخل سرور (نه باز به اینترنت عمومی)

۳ — پیامک (SMS) — ورود با کد OTP

خیلی مهم: بدون پیامک واقعی، کاربر با موبایل خودش وارد نمی‌شود. در حالت تست، کد فقط در گزارش سرور چاپ می‌شود — برای معدن واقعی باید پنل پیامک بخرید.

چه امکاناتی لازم است؟

  • خط ارسال پیامک (شماره فرستنده یا خط خدماتی)
  • API برای ارسال خودکار از طرف نرم‌افزار
  • شارژ اعتبار (پرداخت به ازای هر پیامک)
  • تأیید متن پیامک OTP از سامانه (گاهی متن باید از قبل ثبت شود)
  • پشتیبانی ارسال به همه اپراتورهای ایران (همراه اول، ایرانسل، رایتل)

پیشنهادها — یکی کافی است

گزینه ۱ — کاوه‌نگار (Kavenegar.com)

ثبت‌نام در پنل → احراز هویت → خرید خط → دریافت API Key

پکیج: معمولاً «اعتبار» شارژ می‌کنید + هزینه ماهانه خط؛ بسته پیامک بر اساس مصرف

به تیم فنی: API Key + شماره خط Sender + الگوی متن OTP تأییدشده

گزینه ۲ — فراز اس‌ام‌اس (Farazsms.com / IPPanel)

همان نیازها: Access Key + خط فرستنده

به تیم فنی: API Key + Sender

گزینه ۳ — ملی‌پیامک / پیامک‌پنل / …

هر سرویس ایرانی که API مستند دارد قابل استفاده است — قبل از خرید سالانه، تیم فنی یک پیامک تست بگیرد.

برآورد مصرف: هر ورود ≈ ۱ پیامک. ۱۰۰ ورود در روز ≈ ۳۰۰۰ پیامک در ماه (+ پیامک‌های اعلان اگر فعال کنید).

۴ — اپ موبایل (راننده و خانوار)

چه امکاناتی لازم است؟

  • حساب توسعه‌دهنده برای انتشار اپ اندروید در ایران
  • شماره تماس پشتیبانی برای نمایش در اپ (مثلاً دفتر معدن)

کافه‌بازار (CafeBazaar.ir) — پیشنهاد اصلی ایران

ثبت‌نام توسعه‌دهنده → تأیید هویت → ساخت برنامه → آپلود APK (تیم فنی انجام می‌دهد)

به تیم فنی: دعوت به پنل توسعه‌دهنده یا فرآیند امضای رسمی APK که شما تأیید می‌کنید

Google Play — اختیاری

فقط اگر دسترسی بین‌المللی و حساب توسعه‌دهنده گوگل دارید. برای کاربران داخل ایران کافه‌بازار مهم‌تر است.

۵ — باسکول: آیا برای خودکار شدن باید دستگاه جدید خرید؟

راننده خودش وزن را در گوشی وارد نمی‌کند — این عمدی است تا کسی نتواند وزن را جعل کند. کسی که مسئول باسکول است (یا ادمین) در پنل وب وزن خالی و پر را ثبت و تأیید می‌کند. از همان لحظه برای سیستم آن وزن رسمی است و روی پول و سهم جامعه اثر می‌گذارد.

پس برای شروع، همان باسکولی که دارید کافی است — فقط باید اپراتور آموزش ببیند پنل را باز کند. نیازی به خرید باسکول نو نیست.

«خودکار» یعنی عدد روی نمایشگر بدون تایپ دوباره به سیستم برسد. سه راه دارید:

سطحخرید سخت‌افزار؟توضیح
۱ — همین الان خیر اپراتور عدد را از صفحه باسکول می‌خواند و در پنل وب می‌زند. ساده و برای فاز اول کافی است.
۲ — قدم بعدی (رایج) معمولاً نه کنار باسکول یک PC با نرم‌افزار سازنده است. یک برنامه کوچک (Agent) وزن را به سرور می‌فرستد. قبلش از فروشنده باسکول بپرسید: «API یا خروجی شبکه دارید؟»
۳ — پیشرفته گاهی بله اتصال کابلی مستقیم به indicator — وابسته به برند؛ گران‌تر و پیچیده‌تر.
کار شما الان: نام فروشنده/نصاب باسکول + مدل دستگاه + یک تماس: «می‌شود وزن را خودکار به سیستم فرستاد؟»
به تیم فنی: عکس از نرم‌افزار PC باسکول + هر دفترچه فنی
تا جواب نیامده، همان ثبت دستی در پنل ادامه پیدا کند.

۶ — چیزهایی که الان لازم نیست — بعداً اگر خواستید

Firebase (اعلان لرزشی روی موبایل)

سرویس گوگل است برای وقتی اپ بسته است یک نوتیف بیاید («مأموریت جدید» و …). برای ورود با کد استفاده نمی‌شود — ورود با همان پیامک ایرانی است.

در ایران دسترسی به سرویس‌های بین‌المللی گاهی ناپایدار است. برای نسخه اول پیشنهاد می‌کنیم Firebase نخرید. اعلان مهم را یا با پیامک بفرستید، یا کاربر اپ را باز کند و لیست اعلان‌ها را ببیند (این بخش در سیستم هست).

اگر بعداً خواستید: حساب Firebase (رایگان تا سقف مشخص) + اپ در Firebase ثبت شود — تیم فنی راه می‌اندازد. وابسته کردن روزمره معدن به VPN منطقی نیست.

فضای ابری برای فایل (رسید PDF، عکس)

الان برای «پرداخت شد» در پنل یک لینک وارد می‌شود. بعداً می‌توانید از ابر آروان، فضای لیارا، یا دیسک سرور برای آپلود واقعی فایل استفاده کنید.

پرداخت خودکار بانک

الان سیستم مبلغ و گزارش می‌دهد؛ واریز از حساب معدن/تعاونی انجام می‌شود و در پنل «ثبت شد + شماره پیگیری» می‌زنید. اتصال خودکار به بانک = قرارداد جدا با بانک + هزینه توسعه — فاز بعد.

Redis / سرور قوی‌تر

وقتی کاربر و مأموریت خیلی زیاد شد — فعلاً سرور ۴–۸ گیگابایت RAM کافی است.

۷ — تسویه و بانک (فرآیند، نه خرید سرویس)

  • سیستم Excel خروجی و وضعیت تسویه می‌دهد
  • شخصی در سازمان شما باید در بانک واقعی واریز کند
  • بعد در پنل: ثبت پرداخت + شماره پیگیری
  • نیازی به «خرید درگاه» برای MVP نیست مگر خودتان بخواهید
