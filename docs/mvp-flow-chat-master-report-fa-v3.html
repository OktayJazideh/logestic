چشم‌انداز

اصل طلایی: هیچ فرآیند رسمی خارج از سیستم معتبر نیست — مأموریت، تأیید، تسویه، عضویت، اعتراض، باسکول.

پلتفرم = Logistics + Settlement + Cooperative OS + Governance. جهت ارائه به ذی‌نفعان: زیرساخت تعاونی صنعتی (Industrial Cooperative Infrastructure) / حاکمیت مبتنی بر قانون، audit و شفافیت — نه صرفاً «اپ حمل». کارفرما فقط نیاز عملیاتی ثبت می‌کند؛ پلتفرم Dispatch می‌کند؛ باسکول/مشاور measurement معتبر می‌دهند؛ سیستم پرداخت عملیاتی (مالک + کارمزد پلتفرم) و مشارکت اجتماعی تن‌محور (§۱۴) را جدا محاسبه و تسویه می‌کند.

مقیاس محصول (نه پروژه تک‌معدنی): هدف انتشار پلتفرم ملی/جهانی — هر معدن و هر تعاونی tenant مستقل با قرارداد و Rate Card خودش. پایلوت طلای تفتان فقط اولین استقرار واقعی برای UAT است؛ معماری از روز اول Multi-Mine / Multi-Cooperative (Workspace، scope، Service Contract per طرفین). فاز ۱ محدودیت نوع سرویس است (فعلاً حمل)، نه محدودیت تعداد معادن یا تعاونی‌ها.
اولویت اجرای واقعی (کارفرما — پس از بالا آمدن MVP): دقت تسویه، Audit، Permission، خطاها و edge caseها، Backup، Logging، تأیید داده، Performance — پیش از UI تزئینی یا فیچرهای جانبی غیرضروری.

مسیر طلایی (طبق کارفرما): نیاز کارفرما → تخصیص → اجرا → باسکول/ساعت → تأیید مرجع → محاسبه → تراکنش‌ها → نمایش → تسویه ادمین.

۱۸.۲ تفکیک حسابداری (Operational / Platform / Community Restricted)
